# Security Policy

## Supported versions

This repository is continuously maintained on the `main` branch.
Security fixes are delivered through pull requests and releases.

## Reporting a vulnerability

Please report vulnerabilities privately.

1. Open a security advisory in GitHub Security Advisories for this repository.
2. Include reproduction steps, impact, and affected files.
3. Provide a safe proof of concept when possible.

Do not open public issues for unpatched vulnerabilities.

## Response targets

- Initial triage: within 5 business days
- Remediation plan: as soon as impact is confirmed
- Public disclosure: after a fix is available
